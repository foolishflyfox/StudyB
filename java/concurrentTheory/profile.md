# Java 并发

## 理论基础

CPU、内存、IO 设备的速度有极大的差异，为了合理利用 CPU 的高性能，平衡三者的速度差异，计算机体系结构、操作系统、编译程序都作出了贡献。主要体现为：
- CPU 增加了缓存，以平衡与内存的速度差异；（导致了可见性问题）
- 操作系统增加了进程、线程，以分时复用 CPU，进而均衡 CPU 与 I/O 设备的速度差异；（导致原子性问题）
- 编译程序优化指令次序，使得缓存能够得到更合理地利用；（导致有序性问题）
